00 · Foundation

Design Principles

Every decision in this system is grounded in four discovery inputs: user interviews, project audit findings, competitor/market research, and academic literature on bicycle advocacy and community access.

🔍

Research Foundation

This system responds directly to findings from 3 user interviews (Michael Lee, Leo Fan, Roderick McClain), a competitive analysis of Banana Boat, Sunny D, Gregg's Cycle, Montlake Bicycle Shop, and Mello Fellos, a site audit identifying 7 structural pain points, and 14 peer-reviewed papers on cycling equity and urban mobility.

Mission Bike Works promotes the bicycle as a vehicle for change to empower youth and build resilient communities.
Vision Mobilized people. Thriving communities. Healthy planet.
Values Bicycling · Access · Youth · Environment · Community · Social Justice · Education
♿

Accessibility First

The existing site's yellow background failed accessibility tests (Roderick McClain, internal). This system enforces WCAG AA contrast minimums on every color pairing. Yellow is used as an accent only — never as a large background behind body text.

🗺️

Clear Hierarchy

Three of three interviewees struggled to understand Bike Works' identity and offerings on first visit. This system establishes a single, scannable type scale and strict content structure so new visitors can orient immediately.

🤝

Community Authentic

Research shows bike advocacy organizations must build trust with underserved communities through genuine cultural messaging. The visual system retains Bike Works' bold, energetic identity while elevating professionalism and readability.

🔬Design Rationale
Problem Solved
The original site used yellow as a full-page background, which Roderick McClain (Comms & Grants Manager) flagged as failing accessibility tests. Leo Fan noted that while "color contrast works, extended reading is tiring."
"Bold yellow background will likely fail accessibility tests, wants accessibility and readability." — Roderick McClain, Interview
Yellow as Accent Only
Yellow (#F5C518) is retained as a brand signal and CTA accent — never as a background behind long-form text. This follows the Mello Fellos approach noted in market research: "yellow is used sparingly as an accent color to draw attention to buttons or key elements."
Blue as Primary
BW Blue (#2B63C7) passes WCAG AA on white and is the dominant interactive color. This matches Montlake Bicycle Shop's successful use of navy blue as a "distinct path for users to press the call to action button."
Navy for Depth
Navy (#0D1F4E) creates strong hierarchical anchors — hero backgrounds, stat cards, the navigation — without relying on pure black, keeping the palette warm and community-oriented.
Competitor Learning
Banana Boat's analysis showed that "secondary colors with hues that compliment blue and yellow give an uplifting feeling" — the off-white and light gray neutrals serve exactly this role, preventing visual fatigue between colored sections.

🔬Design Rationale
Why Barlow Condensed
The existing site uses uppercase, bold display text for navigation and cards — a strong brand pattern worth preserving. Barlow Condensed formalizes this into a consistent display scale that is energetic, space-efficient, and community-facing without feeling corporate.
Readability Problem Solved
Leo Fan found the original site "information was concise but main improvement needed was clearer hierarchy." The strict 7-step type scale (Display → Caption) creates immediate visual priority, so users can scan to what matters for them.
"Felt the site contained a lot of text, though information was concise. Suggested communicating key information faster and more efficiently." — Leo Fan, Interview
Single Font Family
Using Barlow and Barlow Condensed as a superfamily pairing ensures visual cohesion at all sizes. No jarring contrast between display and body — a lesson from Gregg's Cycle's competitor analysis: "consistent font styles and sizes for cards and call to action."
Accessible Line Height
Body text uses 1.7 line-height (above WCAG 1.5x minimum) and 16px base size, directly addressing the readability fatigue Leo Fan and Michael Lee both reported on the original dense text layouts.

🔬Design Rationale
Primary — BW Blue
BW Blue for primary actions mirrors the existing "SUPPORT" button observed in the design assets. Blue on white passes WCAG AA, resolving the contrast concerns Roderick raised. Drop shadow reinforces clickability.
CTA — Yellow on Navy
High-priority conversion actions (Get a Bike, Sign Up, Donate) use yellow — the brand's most energetic color — on navy text for the highest achievable contrast. This is directly inspired by the existing "LEARN MORE" button variant observed in design files.
"Immediately drawn to Free and Low-Cost Services. Found programs like Earn-a-Bike highly valuable." — Leo Fan, Interview
No More Than One Primary CTA
Gregg's Cycle was praised because "it's clear that specific types of cards call out to shopping options." Each page section should have a single dominant action — this hierarchy (Primary → CTA → Secondary → Ghost) enforces that.
44px Minimum Touch Target
All button sizes meet 44×44px minimum for mobile accessibility, supporting users who may be accessing the site on mobile while biking or on the go — a key Bike Works audience behavior.

Form Elements

🔬Design Rationale
2px Border vs Background Fill
The original site's forms blended into the yellow background. This system uses 2px bordered inputs on white to ensure form fields stand out on any background color — directly solving the challenge noted in the project discovery: "Needs more contrast for form blocks because it blends in too much with the yellow."
Blue Focus Ring
A 3px blue focus ring (rgba 15% opacity spread) makes keyboard navigation clearly visible, supporting diverse users including those with motor impairments — aligning with Bike Works' equity mission of serving communities "historically excluded."
Zip Code Validation
The Bikes-for-All program has geographic eligibility (SE Seattle, South King County zip codes). Inline error messages on zip fields reduce application friction for eligible low-income users who may be less tech-confident.
"The program prioritizes youth and adults from low-income households and communities that have been historically underserved." — Bikes-for-All Program Desc.
Newsletter Opt-In
The site audit revealed every page has a newsletter sign-up. The checkbox component ensures this critical touchpoint is accessible and clearly labelled, not buried or auto-checked.

🔬Design Rationale
Consistent Card Structure
The site audit found "there isn't a consistent content structure throughout the website." Program cards enforce a rigid 4-zone layout (image, title, description, CTA) so all programs are equally scannable regardless of content volume.
Age Badge as First Signal
Michael Lee's interview revealed users struggled to understand what Bike Works offers. The age badge (Ages 13–18, Adults 18+) on every program card resolves this immediately — users self-select within 2 seconds of seeing a card, before reading any body copy.
"As an organization, it's hard to understand what they are... should provide summary of statement stating who they are and what services they provide." — Michael Lee, Interview
Stat Cards — Impact at a Glance
Academic literature on cycling equity shows organizations must communicate tangible impact to build trust with underserved communities. Large-number stat cards make "300+ bikes annually" and "FREE" immediately legible for users scanning quickly.
Hover Lift Effect
Cards lift (+2px translateY, deeper shadow) on hover — a subtle affordance that communicates interactivity without distracting from content. Mello Fellos was praised for "simple, clear call-to-action" banners; this pattern extends that clarity to every card.
Alert Cards for Key Policies
The sliding scale payment policy is a core equity mechanism. Alert cards ensure this information is surfaced at every relevant touchpoint — not buried in body text — supporting equitable access for users who may disengage if cost is unclear.

🔬Design Rationale
Single Navigation Bar
The project discovery identified that "the website contains two nav bars, which could confuse the user and their journey." This system consolidates into one bar — primary links, icon utilities (phone, map, search), and the Support CTA — eliminating the dual-nav confusion entirely.
"The website contains two nav bars, which could confuse the user and their journey." — Project Discovery Audit
Support as Always-Visible CTA
The "Support" button remains persistently visible in the nav on every page — matching the existing design pattern but now in high-contrast Yellow/Navy. Roderick confirmed donation/volunteer promotion is critical to the org's sustainability.
Yellow Active State
Active nav links use yellow — the most visually distinctive color in the palette — so users always know where they are. This addresses Leo Fan's feedback about difficulty orienting on the site.
Tabs for Long Pages
The site audit noted "Staff and Board pages are excessively long and text-heavy." Tabs break this pattern by segmenting content (Overview, Eligibility, Schedule, FAQ) into digestible chunks while keeping everything accessible on one URL.
Icon-Only Utilities
Phone, map, and calendar icons are compressed to icon-only in the nav (matching the existing design system's icon set) to preserve space for the primary navigation links while keeping utility access a single tap away.

🔬Design Rationale
FREE Badge — Top Priority
Leo Fan was "immediately drawn to Free and Low-Cost Services" and found it "highly valuable." The navy/yellow FREE badge is the most visually prominent badge in the system, ensuring this key differentiator is surfaced everywhere it applies.
"Immediately drawn to Free and Low-Cost Services. Found programs like Earn-a-Bike, virtual repair resources, and open shop access highly valuable." — Leo Fan, Interview
Multi-Audience Filtering
Bike Works serves youth (13–18), adults, volunteers, and donors simultaneously. Filter tags allow each visitor type to quickly surface relevant programs without scrolling through content meant for others — directly reducing the "overwhelming amount of information" problem identified in the audit.
Limited Spots Warning
Roderick noted that giveaway promotions risk "too many signups" which "could disappoint people." The Warning badge surfaces capacity limits early in the browsing experience, managing expectations before users commit to the registration flow.

🔬Design Rationale
4px Base Grid
The site audit directly identified "a lot of uneven spacing in the left side" and "footer components don't have a lot of spacing between the links." A strict 4px base unit (4, 8, 12, 16, 24, 32, 48, 64, 96) eliminates all ad-hoc pixel values and creates the visual rhythm the original site lacked.
"Footer components don't have a lot of spacing between the links. There is a lot of uneven spacing in the left side too." — Project Discovery Audit
Generous Section Spacing
Section-level spacing uses 64px (--space-8) and 96px (--space-9) to create clear visual breathing room between content blocks. This prevents the "overwhelming amount of information" feeling by using whitespace as an organizational tool rather than packing content edge-to-edge.
Radius Philosophy
Smaller radii (4–8px) on interactive elements (buttons, inputs) communicate precision and reliability. Larger radii (12–20px) on content containers (cards, sections) feel warm and approachable — matching Bike Works' community-first identity rather than a cold, corporate aesthetic.
